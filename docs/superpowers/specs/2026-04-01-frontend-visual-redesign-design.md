# 前端视觉风格重设计方案

**日期**：2026-04-01
**类型**：设计规范
**优先级**：视觉风格优先

---

## 概述

OpenCode Agent 平台前端目前采用深色主题，整体视觉风格偏暗、不够商务化。本方案旨在将前端改造为明亮清晰的现代企业风格，提升专业性和可用性，同时保持现有功能不变。

### 改造范围

- **视觉风格升级**：从深色主题改为灰白现代企业风格
- **配色系统重构**：建立统一的色彩体系
- **布局优化**：采用宽松卡片式布局
- **交互细节改进**：统一间距、圆角、阴影、动画

### 不涉及内容

- 功能新增或修改（Skill 管理、MCP 管理等功能保持现状）
- 后端 API 变更
- 组件结构重构

---

## 设计目标

1. **专业性**：采用灰白基调 + 低饱和蓝色点缀，呈现现代商务风格
2. **清晰性**：明亮背景 + 合理对比度，提升可读性
3. **一致性**：建立统一的设计系统，所有页面遵循相同规范
4. **可维护性**：通过 CSS 变量实现，便于后续调整

## 设计方案

### 1. 色彩系统

#### 1.1 基础色板

**背景色系**：
- 主背景：`#f8f9fb`（极浅的灰白，类似 Notion）
- 次级背景/卡片：`#ffffff`（纯白，用于卡片、面板）
- 悬停背景：`#f1f3f7`（略深的灰，用于交互反馈）
- 边框色：`#e1e5eb`（浅灰边框，清晰但不突兀）

**文字色系**：
- 主文字：`#1f2937`（深灰，不是纯黑，更柔和）
- 次级文字：`#6b7280`（中灰，用于说明文字、次要信息）
- 禁用/辅助文字：`#9ca3af`（浅灰，用于占位符等）

**强调色系（低饱和蓝色）**：
- 主强调色：`#4a90e2`（类似 Notion 的蓝色，稳重不刺眼）
- 悬停强调：`#3b82f6`（略深，用于按钮悬停等）
- 强调色背景：`#e8f4fd`（极浅蓝，用于选中状态、高亮背景）

**语义色系**：
- 成功：`#10b981`（绿色）
- 警告：`#f59e0b`（橙色）
- 错误：`#ef4444`（红色）

#### 1.2 CSS 变量映射

```css

  --bg-secondary: #ffffff;

  --border: #e1e5eb;

  /* 文字 */

  --text-muted: #6b7280;

  --accent-hover: #3b82f6;
  --accent-bg: #e8f4fd;

  /* 消息气泡 */
  --user-bg: #e8f4fd;

:root[data-theme='dark'] {
  /* 保留深色模式变量，便于未来切换 */
  --bg: #0f1117;
  --bg-secondary: #171b26;
  --bg-hover: #222736;
  --panel: rgba(20, 24, 35, 0.88);
  --border: #2a3040;
  --text: #e5e7eb;
  --text-muted: #94a3b8;
  --accent: #3b82f6;
  --accent-hover: #2563eb;
  --user-bg: #1e3a5f;
  --assistant-bg: #1a1f2d;
}
```

### 2. 布局与间距系统

#### 2.1 间距系统（采用 8px 基准）

统一使用 8px 作为基础单位，所有间距都是 8 的倍数：

- **极小间距**：`4px` — 用于紧凑元素内部（按钮内图标和文字间距）
- **小间距**：`8px` — 用于标签、小按钮之间的间距
- **基础间距**：`16px` — 用于卡片内部元素的间距，内容段落间距
- **中等间距**：`24px` — 用于卡片之间、页面区块之间的间距
- **大间距**：`32px` — 用于页面顶部与内容之间、重要区块之间的间距
- **超大间距**：`48px` — 用于页面边距、主要区块分隔

#### 2.2 圆角系统

采用现代柔和的圆角：

- **小圆角**：`8px` — 用于按钮、小卡片、标签
- **基础圆角**：`12px` — 用于标准卡片、输入框、下拉框
- **大圆角**：`16px` — 用于大面板、模态框
- **超大圆角**：`20px` — 用于页面级卡片、hero 区域

#### 2.3 卡片样式

统一的卡片样式，增加视觉层次感：

```css
.card {
  background: var(--bg-secondary);      /* 白色背景 */
  border: 1px solid var(--border);      /* 浅灰边框 */
  border-radius: 16px;                  /* 大圆角 */
  padding: 24px;                        /* 中等间距 */
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.04); /* 极浅阴影 */
}

.card:hover {
  box-shadow: 0 4px 16px rgba(0, 0, 0, 0.08); /* 悬停时阴影加深 */
  border-color: var(--accent);          /* 边框变蓝 */
}
```

#### 2.4 页面布局调整

**顶部导航栏**：
- 高度：从 `79px` 减少到 `64px`
- padding：`16px 24px`
- 背景：纯白 + 极浅阴影，sticky 定位

**内容区域**：
- 页面边距：`32px` 或 `48px`
- 内容最大宽度：卡片区域建议 `max-width: 1200px`

**侧边栏（Sessions 页面）**：
- 宽度：保持 `280px`
- 背景：浅灰 `#f8f9fb`
- 卡片间距：`8px`

### 3. 字体与排版系统

#### 3.1 字体族选择

**主字体**：
```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "Roboto",
  "Helvetica Neue",
  Arial,
  sans-serif;
```

**代码/等宽字体**：
```css
font-family:
  ui-monospace,
  "SF Mono",
  "Cascadia Code",
  "Roboto Mono",
  Consolas,
  monospace;
```

#### 3.2 字号系统

- **页面标题（H1）**：`28px - 32px`
- **区块标题（H2）**：`20px - 24px`
- **卡片标题（H3）**：`16px - 18px`
- **正文内容**：`14px - 16px`
- **辅助文字**：`12px - 14px`
- **小标签/徽章**：`11px - 12px`

#### 3.3 字重使用

- **正常文字**：`400`
- **标题/强调**：`500 - 600`
- **极轻文字**：`300`（仅用于装饰性超大标题）

#### 3.4 行高规范

- **标题行高**：`1.2 - 1.3`
- **正文行高**：`1.5 - 1.6`
- **代码/技术内容**：`1.4`

### 4. 页面改造细节

#### 4.1 顶部导航栏

**改造要点**：
- 背景：从 `rgba(15, 17, 23, 0.72)` 改为纯白 `#ffffff` + 浅阴影

- Logo 区域："OC" brand mark 保持蓝色渐变但更明亮
- 导航链接：灰色文字，悬停时背景 `#f1f3f7`，激活时背景 `#e8f4fd` + 蓝色文字
- 主题切换按钮：灰色边框，悬停时背景 `#f1f3f7`
- 退出按钮：保持红色悬停效果

#### 4.2 Sessions 页面

**Sidebar 改造**：

- 背景：`#f8f9fb`

- 输入区域：白色背景，输入框白色 + 浅灰边框，发送按钮蓝色

#### 4.3 Skills 页面

**改造要点**：
- 页面背景：从深色渐变改为浅灰 `#f8f9fb` + 极浅的蓝色渐变装饰
- Hero 区域：白底卡片，标题深灰，副标题中灰

  - 白色背景 + 浅灰边框 + 16px 圆角

  - 标签：浅蓝背景 `#e8f4fd` + 蓝色文字
  - 状态徽章：启用用浅绿背景 + 绿色文字，禁用用浅灰背景 + 灰色文字
- 模态框：白色背景，标题深灰，内容区浅灰分隔

#### 4.4 Settings 页面

**改造要点**：
- 页面背景：`#f8f9fb`

  - 白色背景 + 浅灰边框 + 20px 圆角

#### 4.6 Schedule 页面

### 5. 响应式设计与交互细节

#### 5.1 响应式断点

- **桌面端**：`≥ 920px` — 保持当前布局
- **平板端**：`720px - 919px` — Sidebar 变为可折叠，顶部导航简化
- **移动端**：`< 720px` — 单列布局，卡片全宽

#### 5.2 过渡动画

采用简洁流畅的过渡：

- **颜色过渡**：`0.15s - 0.2s ease`（用于背景、边框、文字颜色变化）
- **阴影过渡**：`0.2s ease`（用于卡片悬停效果）
- **位移过渡**：`0.25s - 0.3s ease`（用于展开/折叠、弹出等）

示例：
```css
.card {
  transition:
    background-color 0.15s ease,
    border-color 0.15s ease,
    box-shadow 0.2s ease;
}
```

#### 5.3 阴影层级

定义 3 级阴影：

- **Elevation 1**（卡片默认）：
  ```css
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.04);
  ```

- **Elevation 2**（悬停/弹出）：
  ```css
  box-shadow: 0 4px 16px rgba(0, 0, 0, 0.08);
  ```

- **Elevation 3**（模态框/下拉菜单）：
  ```css
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.12);
  ```

#### 5.4 加载状态

统一加载指示器样式：
- Spinner：使用蓝色 `var(--accent)`，直径 `32px`，线条粗细 `3px`
- 骨架屏：使用浅灰背景 `#f1f3f7`，pulse 动画

#### 5.5 深色模式兼容

- 所有 CSS 变量使用 `:root[data-theme='light']` 和 `:root[data-theme='dark']` 区分
- 用户可以通过顶部导航的"主题切换"按钮切换
- 默认主题改为 `light`

## 实施策略

### 方案选择：渐进式样式替换

**原因**：
- 改动范围小，风险低
- 不影响现有功能
- 实施速度快，1-2 天可完成基础改造

### 实施步骤

1. **替换 CSS 变量**（1-2 小时）
   - 修改 `App.css` 中的 `:root` 变量定义
   - 默认主题改为 `light`

2. **调整全局样式**（2-3 小时）
   - 修改顶部导航栏样式
   - 修改基础组件样式（按钮、输入框、卡片等）

3. **逐页面调整**（4-6 小时）
   - Sessions 页面（sidebar + chat view）
   - Skills 页面
   - Settings 页面
   - Documents 页面
   - Schedule 页面

4. **响应式适配**（2-3 小时）
   - 测试移动端和平板端布局
   - 调整断点样式

5. **测试验证**（1-2 小时）
   - 功能测试：确保所有功能正常
   - 视觉测试：检查所有页面的样式一致性
   - 浏览器兼容性测试

### 预计工作量

**总计**：1-2 个工作日（10-16 小时）
